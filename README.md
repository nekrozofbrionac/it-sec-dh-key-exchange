# Hausarbeit für IT-Sicherheit - Diffie-Hellman Schlüsselaustausch

### Zu machen:

Erklären Sie wie Diffie-Hellman (DH) es zwei Parteien erlaubt
sich auf ein geteiltes Geheimnis zu einigen.

Implementieren und demonstrieren Sie den DH-
Schlüsselaustausch zwischen 3 oder mehr Parteien.

Erweitern Sie die Demo um einen simulierten Man-in-the-Middle-
Angriff (konzeptuell in Ihrer kontrollierten Umgebung) und
erklären Sie welche Voraussetzungen und Auswirkungen dazu
gehören.

Vergleichen Sie die Ergebnisse der Nutzung von authentifizierten
DH und unauthentifizierten DH.

Geben Sie abschließend Erklärungen wie reale Systeme den
Angriff vermeiden und warum Forward Secrecy wichtig ist.
